- List `output/transcripts/` subdirectories.

- Verify: every `.md` filename matches `\d{2}-[a-z0-9-]+\.md` pattern.

**SC2 — Checkpoint resume:**
- Read `output/progress.json`.
- Verify: `completed` array length >= 90.
- Cross-reference: every mdId in progress.json exists in manifest.json.
- NOTE: The actual resume test (re-running pipeline and seeing all-skip) is verified by re-running `node scripts/extract-transcripts.mjs` and checking output contains only `[SKIP]` lines and 0 new fetches. **SC3 — missing-transcripts.log:**
- Read `output/missing-transcripts.log`.
- Verify: file exists and is non-empty.
- Count lines. Verify: >= 21 (one per null-loomId video).
- Verify: every line has 5 tab-separated fields (timestamp, reason, section, title, loomId).
- Cross-reference: count of `null_loom_id` entries matches count of null-loomId videos in manifest.

**SC4 — No rate limiting:**
- Check `output/missing-transcripts.log` for `fetch_failed` entries.
- If 0 `fetch_failed` entries, SC4 passes (all 74 Loom fetches succeeded).
