Create TWO scripts:

**Script 1: `scripts/extract-skool-content.js`** — Browser console script (runs in authenticated Skool session)

This script fetches the individual lesson pages for all 21 null-loomId videos and extracts their content. It follows the same pattern as `scripts/enrich-manifest.js` (browser console, fetch with credentials).

Implementation:
1. First, load the course tree from `__NEXT_DATA__` on the current page (same as enrich-manifest.js).
2. Identify all videos where `loomId` is null (should find 21).
3. For each null-loomId video, fetch its individual lesson page:
   ```javascript
   const resp = await fetch(baseUrl + '?md=' + mdId, { credentials: 'include' });
   const html = await resp.text();
   ```
4. Parse `__NEXT_DATA__` from the response HTML using the regex:
   ```javascript
   const ndMatch = html.match(/<script[^>]+id=["']__NEXT_DATA__["'][^>]*>([\s\S]*?)<\/script>/i);
   ```
5. Extract content from the parsed data. Look for these fields in priority order:
   - `pageProps.selectedModule.course.metadata.body` — lesson body text (HTML or plain text)
   - `pageProps.selectedModule.course.metadata.videoLink` — may be YouTube URL if not Loom
   - If neither exists, also try `pageProps.course.children` tree to find the matching lesson by mdId and check its `metadata.body` and `metadata.videoLink`
   - ALSO: parse the HTML response for visible text content using DOM parsing (create a temporary div, set innerHTML, extract textContent) as a fallback
6. For each video, build a content object:
   ```javascript
   {
     mdId: "...",
     title: "...",
     section: "...",
     contentType: "youtube" | "text" | "empty",
     youtubeUrl: "https://youtube.com/..." or null,
     youtubeId: "..." or null,
     textContent: "..." or null,
     rawVideoLink: "..." or null
   }
   ```
   - `contentType: "youtube"` — if videoLink contains youtube.com or youtu.be
   - `contentType: "text"` — if body text is present and non-trivial (> 20 chars)
   - `contentType: "empty"` — if neither video nor text content is found (true placeholder)
7. Detect YouTube URLs with regex: `/(?:youtube\.com\/(?:watch\?v=|embed\/)|youtu\.be\/)([a-zA-Z0-9_-]{11})/`
8. Add 500ms delay between page fetches to avoid rate limiting.
9. Output the results as a JSON array. Try `copy(JSON.stringify(results, null, 2))` for clipboard, and also `console.log(JSON.stringify(results, null, 2))` as fallback.
10. Print summary: count by contentType (youtube, text, empty).

**Script 2: `scripts/backfill-stubs.mjs`** — Node.js script (runs locally)

Usage: `node scripts/backfill-stubs.mjs`

This script reads `output/skool-content.json` (saved from clipboard after browser script) and overwrites the 21 stub markdown files with real content.

Implementation:
1. Read `output/manifest.json` and `output/skool-content.json`.
2. For each entry in skool-content.json, find the matching video in manifest by mdId. Determine section and video order to compute the file path:
   ```javascript
   const sectionSlug = `${pad(section.order)}-${slugify(section.name)}`;
   const videoSlug = `${pad(video.order)}-${slugify(video.title)}`;
   const filePath = join(TRANSCRIPTS_DIR, sectionSlug, `${videoSlug}.md`);
   ```
   Use the SAME `slugify` and `pad` functions from `extract-transcripts.mjs`.

3. Generate markdown based on contentType:

   **For `contentType: "youtube"`:**
   ```markdown
   # {title}

   **Section:** {section.name}
   **Source:** YouTube ({youtubeUrl})

   ## Content

   *YouTube video embed — transcript not available via Loom pipeline.*
   *Watch at: {youtubeUrl}*
   ```
   If a YouTube transcript was fetched (stretch goal — see note below), include it under `## Transcript` instead.

   **For `contentType: "text"`:**
   ```markdown
   # {title}

   **Section:** {section.name}

   ## Content

   {textContent}
   ```

   **For `contentType: "empty"`:**
   ```markdown
   # {title}

   **Section:** {section.name}

   ## Content

   *This lesson is a placeholder — no content has been published yet.*
   ```

4. Write the updated markdown file, overwriting the existing stub.
5. Log each file updated: `[UPDATED] {path} ({contentType})`.
6. Track a separate backfill checkpoint in `output/progress.json` — add a `backfilled` array alongside the existing `completed` array. After each successful backfill, add the mdId to `backfilled` and save atomically (same tmp+rename pattern). On re-run, skip mdIds already in `backfilled`.
7. After all updates, rewrite `output/missing-transcripts.log` to remove the 21 `null_loom_id` entries that now have content, keeping only entries that are genuinely empty placeholders or had `fetch_failed` reason.
8. Print summary: `Done: {N} updated ({N} youtube, {N} text, {N} empty)`.

**NOTE on YouTube transcripts:** Fetching YouTube transcripts programmatically is complex (requires either youtube-transcript-api npm package or scraping). For this plan, capture the YouTube URL at minimum. If the executor finds a simple way to fetch YouTube transcripts (e.g., via a lightweight npm package), include them. Otherwise, the YouTube URL alone is sufficient — Phase 3 summarization can handle YouTube-only entries differently. Do NOT add heavy dependencies just for YouTube transcripts.

IMPORTANT: Reuse the exact `slugify()` and `pad()` functions from `extract-transcripts.mjs` to ensure file paths match exactly. Copy them into `backfill-stubs.mjs` — do NOT import (the extraction script is not structured as a module with exports).
