```javascript
// GraphQL query body — MUST be plain object, NOT array
const body = {
  operationName: 'FetchVideoTranscript',
  variables: { videoId: loomId, password: null },
  query: 'query FetchVideoTranscript($videoId: ID!, $password: String) { fetchVideoTranscript(videoId: $videoId, password: $password) { ... on VideoTranscriptDetails { captions_source_url transcription_status __typename } ... on GenericError { message __typename } __typename } }'
};

// Headers — do NOT include x-loom-request-source (causes 400)
const headers = {
  'Content-Type': 'application/json',
  Accept: 'application/json',
  'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
};

// Response path: json.data.fetchVideoTranscript.captions_source_url
// Check captions_source_url !== null (NOT transcription_status === 'completed')

// VTT parser — strip timestamps, voice tags, cue numbers, HTML tags:
function parseVttToText(vtt) {
  return vtt.split('\n')
    .filter(l => !l.includes('-->') && l.trim() !== '' && !l.match(/^\d+$/) && !l.startsWith('WEBVTT') && !l.startsWith('NOTE'))
    .map(l => l.replace(/<v[^>]*>/g, '').replace(/<\/v>/g, '').replace(/<\/?[a-z][^>]*>/g, '').replace(/^\d+\s+/, '').trim())
    .filter(l => l.length > 0)
    .join(' ').trim();
}
```

  <name>Task 1: Create extract-transcripts.mjs pipeline script</name>
  <files>scripts/extract-transcripts.mjs</files>
  <action>
Create `scripts/extract-transcripts.mjs` — the batch extraction pipeline. This is a single ESM script (~150-200 lines) that:

1. **Reads manifest**: Load `output/manifest.json`, iterate `sections[].videos[]` sequentially.

2. **Progress checkpoint** (EXTR-05):
   - Load `output/progress.json` on startup (create empty `{ completed: [] }` if missing).
   - Use `video.mdId` as checkpoint key (NOT loomId — loomId can be null for multiple videos, see Pitfall 1 in research).
   - After each successful video write, save progress atomically: write to `.tmp` file, then `fs.renameSync` to final path.
   - Secondary check: if output markdown file already exists and is non-empty, skip and add to progress set.

3. **Loom transcript fetch** (EXTR-04):
   - Copy the proven GraphQL query and VTT parser from `scripts/test-loom-batch.mjs` (see interfaces above).
   - POST to `https://www.loom.com/graphql` with plain object body (NOT array — causes 400).
   - Do NOT include `x-loom-request-source` header (causes 400).
   - Check `captions_source_url !== null` as success condition (NOT `transcription_status === 'completed'` — Loom returns "success" not "completed").
   - Fetch VTT from `captions_source_url`, parse to plain text using `parseVttToText`.

4. **Rate limiting** (EXTR-06):
   - 2500ms fixed delay between ALL Loom API calls (not just retries).
   - Exponential backoff on HTTP 429 or 5xx: starting at 5000ms, doubling each retry, cap at 60000ms, max 3 retries per video.
   - Log every retry with attempt number, error, and wait time.

5. **Markdown file writing** (MKDN-01):
   - Template for videos WITH transcript:
     ```
     # {video.title}

     **Section:** {section.name}
     **Loom ID:** {video.loomId}

     ## Transcript

     {transcript text}
     ```
   - Template for videos with null loomId (stub):
     ```
     # {video.title}

     **Section:** {section.name}

     ## Transcript

     *No transcript available: no Loom video ID*
     ```

6. **Folder hierarchy** (MKDN-02):
   - Output base: `output/transcripts/`
   - Section folder: `{padded_order}-{slugified_section_name}/` (e.g., `01-introduction-to-the-course/`)
   - Use `fs.mkdirSync(dir, { recursive: true })` to create.

7. **Sortable filenames** (MKDN-03):
   - Video file: `{padded_order}-{slugified_title}.md` (e.g., `01-introduction.md`)
   - Padding: 2 digits (`String(n).padStart(2, '0')`)
   - Slugify: `toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '')` — no npm library needed.
   - Truncate slug at 80 chars to prevent filesystem issues.

8. **Null-loomId handling**:
   - Write stub markdown (see template above).
   - Log to `output/missing-transcripts.log` with reason `null_loom_id`.
   - Mark as completed in progress.json (so resume doesn't re-process).

9. **Failed fetch handling**:
   - After 3 retries exhausted, log to `output/missing-transcripts.log` with reason `fetch_failed`.
   - Do NOT write a markdown file for failed fetches (leave gap visible).
   - Do NOT mark as completed in progress.json (retry on next run).
   - Continue to next video — do NOT abort pipeline.

10. **missing-transcripts.log format**: Tab-separated, append-only:
    ```
    {ISO timestamp}\t{reason}\t{section.name}\t{video.title}\t{loomId or 'null'}
    ```

11. **Console output**: Log every video's outcome:
    - `[SKIP]` for already-completed
    - `[STUB]` for null-loomId stub writes
    - `[OK]` for successful fetch with word count
    - `[FAIL]` for failed after retries
    - Final summary line: `Done: {N} fetched, {N} skipped, {N} stubs, {N} failed`

12. **Exit codes**: 0 = all succeeded or skipped, 1 = any failed fetches (so CI can detect gaps).

IMPORTANT: Use `import fetch from 'node-fetch'` (ESM). Use `import { readFileSync, writeFileSync, ... } from 'fs'`. Use `import { join, dirname } from 'path'`. Use `import { fileURLToPath } from 'url'` for `__dirname` equivalent.
