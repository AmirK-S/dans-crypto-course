<objective>
Build and run the AI summarization pipeline that generates per-section trading playbook summaries and a master course-level summary using hierarchical AI API calls.

Purpose: This is the core deliverable of Phase 3 — transforming raw transcripts into an actionable trading playbook. Section summaries use AI model (Haiku) (cost-optimized extractive tasks), and the master summary uses AI model (Sonnet) (higher synthesis quality). Anti-hallucination prompt engineering ensures only content explicitly stated in transcripts appears in summaries.

Output: `scripts/generate-summaries.mjs` (pipeline script), 12 `SECTION_SUMMARY.md` files (one per section folder), `output/MASTER_SUMMARY.md` (course-level synthesis).
</objective>

<task type="auto">
  <name>Task 1: Create the summarization pipeline script</name>
  <files>scripts/generate-summaries.mjs</files>
  <action>
  Create `scripts/generate-summaries.mjs` — a Node.js ESM script that generates per-section and master summaries using the AI API. Follow the research patterns exactly (03-RESEARCH.md Pattern 1-7).

  **Structure:**

  1. **Constants at top of file:**
     - `SECTION_MODEL = 'ai-model-haiku'`
     - `MASTER_MODEL = 'ai-model-sonnet'`
     - `INTER_SECTION_DELAY_MS = 1000`
     - `TRANSCRIPTS_DIR = join(ROOT, 'output', 'transcripts')`
     - `MASTER_SUMMARY_PATH = join(ROOT, 'output', 'MASTER_SUMMARY.md')`

  2. **API key guard (fail-fast):**
     ```javascript
     if (!process.env.AI_API_KEY) {
       console.error('ERROR: AI_API_KEY not set. Run: export AI_API_KEY=sk-ant-...');
       process.exit(1);
     }
     ```

  3. **`cleanTranscript(rawText)` function:**
     Same filler/artifact removal as `clean-transcripts.mjs` (from Plan 03-01). This is a safety net for any content not caught by in-place cleaning. Apply all regex patterns from research Pattern 4.

  4. **`getSectionTranscripts(sectionDir)` function:**
     - Read all `.md` files in the section dir, excluding `SECTION_SUMMARY.md`.
     - Sort by filename (numeric prefix order).
     - For each file:
       - Skip if content contains "YouTube video embed" (YouTube stubs — no usable text).
       - Skip if word count < 100 (Skool navigation boilerplate, stubs).
       - Extract text after `## Transcript` or `## Content` heading via regex: `/## (?:Transcript|Content)\s*\n+([\s\S]+)/`
       - If match found and match[1].trim().length > 50, push `cleanTranscript(match[1].trim())` to array.
     - Return array of cleaned transcript strings (empty array = skip this section).

  5. **`generateSectionSummary(sectionName, transcriptTexts, client)` function:**
     - Wrap each transcript in `<transcript index="N">` XML tags.
     - Call `client.messages.create()` with:
       - `model: SECTION_MODEL`
       - `max_tokens: 4096`
       - `temperature: 0` (extractive, deterministic)
       - System prompt: trading analyst extracting structured knowledge. CRITICAL RULES: only include EXPLICITLY STATED info, do NOT fabricate numbers/percentages/indicators/thresholds, do NOT infer or extrapolate, if a section has no explicit content write "Not explicitly covered in this section's transcripts.", quote or paraphrase only what instructor actually says.
       - User message: asks to summarize the transcripts from `sectionName` into trading playbook format with these exact sections:
         - `### Core Concepts` (3-7 bullets)
         - `### Rules` (explicit rules/principles, instructor's exact phrasing)
         - `### Entry Signals` (entry criteria, conditions, triggers — omit if not covered)
         - `### Position Sizing / Risk Management` (explicit guidance — omit if not covered)
         - `### Key Warnings / What Not To Do` (explicit "do not do this" guidance)
       - Ends with: "Do not add information that is not in the transcripts. Do not preamble."
     - Return `message.content[0].text`.
     - Use the EXACT prompt text from research Pattern 2 (03-RESEARCH.md lines 109-162).

  6. **`generateMasterSummary(sectionSummaries, client)` function:**
     - Wrap each section summary in `<section name="...">` XML tags.
     - Call `client.messages.create()` with:
       - `model: MASTER_MODEL`
       - `max_tokens: 8192`
       - System prompt: trading analyst synthesizing complete crypto trading course. CRITICAL RULES: only include info present in section summaries, no external knowledge, if sections conflict note the conflict, preserve instructor's original phrasing.
       - User message: asks to synthesize section summaries into master trading playbook with:
         - `## Course Philosophy`
         - `## The Strategy in Brief`
         - `## Master Rules (from all sections)` (with section attribution)
         - `## Entry Signals & Conditions` (with attribution)
         - `## Position Sizing & Risk Management` (with attribution)
         - `## What NOT To Do (Do-Not-Do List)`
         - `## Section-by-Section Quick Reference` (one-line per section)
       - Ends with: "Do not preamble. Do not add information not in the provided summaries."
     - Return `message.content[0].text`.
     - Use the EXACT prompt text from research Pattern 3 (03-RESEARCH.md lines 166-229).

  7. **`main()` function — sequential pipeline:**
     - Read section directories from TRANSCRIPTS_DIR, sorted alphabetically.
     - For each section:
       a. Check if `SECTION_SUMMARY.md` exists → skip (resume/checkpoint).
       b. Call `getSectionTranscripts()`.
       c. If empty → write stub: `*No transcript content available — videos in this section are hosted on YouTube or have no Loom transcript.*` → continue.
       d. Log: `[SUMMARIZE] sectionName (N transcripts)`.
       e. Call `generateSectionSummary()`.
       f. Validate: if summary.length < 200, log warning (suspiciously short).
       g. Write `SECTION_SUMMARY.md` in the section folder.
       h. Collect `{ name, summary }` for master synthesis.
       i. Wait `INTER_SECTION_DELAY_MS`.
     - After all sections: check if `MASTER_SUMMARY.md` exists → skip. Otherwise call `generateMasterSummary()`, validate length, write file.
     - Print final summary: sections generated, sections skipped, master status.

  8. **Error handling:**
     - Wrap each section call in try/catch. On error, log `[ERROR] sectionName: error.message` and continue to next section (do not abort entire pipeline for one section failure).
     - Wrap master summary call in try/catch. On error, log and exit with code 1.
     - If API returns a response but `message.content` is empty or not an array, log warning and skip.

  IMPORTANT: Do NOT use prefilled assistant messages (`{role: "assistant", content: "..."}`) — deprecated for AI 4.5+ models.
  IMPORTANT: Do NOT process section 12 ("Putting Everything Together") — no folder exists for it.
  </action>

  </done>
