<task type="auto">
  <name>Task 1: Create browser console enrichment script for per-lesson section names and loomIds</name>
  <files>scripts/enrich-manifest.js</files>
  <action>
Create `scripts/enrich-manifest.js` as a self-contained browser console script (IIFE, no imports) designed to be pasted into the browser DevTools console on any authenticated Skool classroom page.

**Why this script is needed:** The 01-01 extraction script ran on the classroom root page, which rendered 95 lesson links but did NOT render section headings or Loom embed iframes. The verification report (01-VERIFICATION.md) confirmed all 95 videos have "Unknown Section" and 94/95 have null loomId. Per-lesson page navigation is required to extract these from `__NEXT_DATA__`.

**The script MUST:**

1. **Accept the existing manifest as input.** At the top of the script, define a `MANIFEST` constant that the user will paste the contents of `output/manifest.json` into. Use a placeholder comment:
   ```javascript
   const MANIFEST = /* PASTE output/manifest.json CONTENTS HERE */;
   ```
   Alternatively (and preferably), use `prompt('Paste manifest JSON:')` to accept it at runtime if the manifest is small enough. Since the manifest is ~95 entries (~30KB), prompt() may truncate. Instead, instruct the user to first run:
   ```javascript
   window.__MANIFEST = JSON.parse(/* paste manifest JSON string here */);
   ```
   Then the enrichment script reads from `window.__MANIFEST`.

   Actually, the SIMPLEST approach: the script fetches each lesson page using the SAME base URL pattern and collects data from scratch. It already has all 95 mdIds from `output/manifest.json`. So the script should:
   - First, collect all `a[href*="md="]` links from the current page (same as extract-skool.js) to get all mdIds
   - Then, for each mdId, fetch `${baseUrl}?md=${mdId}` using `fetch()` (same-origin, cookies included automatically)
   - Parse the response HTML to extract `__NEXT_DATA__` JSON
   - From `__NEXT_DATA__`, extract: `metadata.videoLink` (Loom embed URL with loomId) and the section/module name

2. **Extract section name from each lesson page's `__NEXT_DATA__`:**
   - Parse the fetched HTML to find `<script id="__NEXT_DATA__">` content
   - Walk the `__NEXT_DATA__` JSON tree looking for the lesson's module/section context
   - Look for these paths (try in order):
     a. `props.pageProps` -> look for a `module` or `section` or `category` object with a `name` or `title` field
     b. Breadcrumb data: look for an array of navigation items (breadcrumb trail) that includes the section name as an ancestor
     c. The `curriculum` or `modules` array where each module has a `name` and nested `pages`/`lessons` — find which module contains the current lesson's mdId
   - If none of these paths work, try parsing the HTML for a visible breadcrumb element (e.g., a nav element or `.breadcrumb` class)
   - As a FINAL fallback, extract from the page title or heading if it includes a section prefix
   - Log which extraction strategy succeeded so the user can verify

3. **Extract loomId from each lesson page:**
   - From `__NEXT_DATA__`: look for `metadata.videoLink` (confirmed pattern from research, e.g., `"https://www.loom.com/embed/<32-char-hex>?..."`)
   - Extract the 32-char hex ID using regex: `/\/embed\/([a-f0-9]{32})/i`
   - If `metadata.videoLink` is absent, check for iframe elements in the HTML: look for `loom.com/embed` in the raw HTML string
   - If neither found, set loomId to null (video may not have a Loom embed — could be text-only or use a different player)

4. **Rate-limit the page fetches:**
   - Add a configurable delay between requests: `const DELAY_MS = 500;` (500ms between each fetch — 95 pages = ~48 seconds total)
   - Use `async/await` with a `sleep(ms)` helper: `const sleep = ms => new Promise(r => setTimeout(r, ms));`
   - Log progress every 10 lessons: `[enrich] 10/95 done... (current section: "Section Name")`
   - This runs within the same authenticated browser session, so cookies are included automatically in same-origin fetch() calls

5. **Build the enriched manifest:**
   - Group videos by the extracted section name
   - Assign section order by the order sections first appear (top to bottom based on the original manifest ordering)
   - Assign video order within each section sequentially
   - Preserve the original manifest structure:
     ```json
     {
       "extractedAt": "<new ISO timestamp>",
       "enrichedAt": "<new ISO timestamp>",
       "courseUrl": "<base classroom URL>",
       "totalVideos": <count>,
       "sections": [
         {
           "name": "<real section name>",
           "order": 1,
           "videos": [
             {
               "title": "<video title>",
               "loomId": "<32-char hex or null>",
               "loomEmbedUrl": "<full embed URL or null>",
               "order": 1,
               "mdId": "<32-char hex>"
             }
           ]
         }
       ]
     }
     ```

6. **Output to clipboard** using `copy(JSON.stringify(enrichedManifest, null, 2))`.

7. **Log a detailed summary to console:**
   ```
   === ENRICHMENT COMPLETE ===
   Total lessons processed: 95
   Sections found: <N> (expected ~13)
   Videos with loomId: <N>/95
   Videos without loomId: <N>/95

   Section breakdown:
     1. <Section Name> - <N> videos, <N> with loomId
     2. <Section Name> - <N> videos, <N> with loomId
     ...

   Section extraction method: <which __NEXT_DATA__ path worked>

   Manifest copied to clipboard.
   Paste into output/manifest.json to save.
   ```

8. **Error handling:**
   - If a page fetch fails (network error, 4xx), log the error and continue to the next lesson. Set that lesson's section and loomId to null.
   - Track failed fetches and report them in the summary.
   - If ALL page fetches fail, log: "All fetches failed. You may have been logged out or rate-limited. Try again after reloading the page."

**Important design notes:**
- The script runs in the browser console on an authenticated Skool page. All fetch() calls are same-origin and include the session cookie automatically.
- The script does NOT navigate the browser — it uses `fetch()` to get each page's HTML as a string, then parses `__NEXT_DATA__` from the HTML string using regex: `/<script id="__NEXT_DATA__"[^>]*>([\s\S]*?)<\/script>/`
- This is a single-paste script with NO external dependencies.
- Do NOT use `document.getElementById('__NEXT_DATA__')` (that only works on the CURRENT page, not fetched pages).
- Keep the script under 200 lines if possible.
