  <action>
Create `scripts/validate-manifest.mjs` â€” a Node.js ESM script that reads `output/manifest.json` and validates its structure, completeness, and data quality.

The script MUST check:

1. **File exists and is valid JSON:** Read `output/manifest.json`, parse as JSON. Exit 1 with clear error if missing or malformed.

2. **Top-level structure:**
   - Has `extractedAt` (string, ISO date format)
   - Has `courseUrl` (string, contains "skool.com")
   - Has `totalVideos` (number, > 0)
   - Has `sections` (array, non-empty)

3. **Section structure** (for each section):
   - Has `name` (string, non-empty)
   - Has `order` (number, >= 1)
   - Has `videos` (array, non-empty)
   - Section orders are sequential starting from 1 with no gaps

4. **Video structure** (for each video in each section):
   - Has `title` (string, non-empty)
   - Has `mdId` (string, matches `/^[a-f0-9]{32}$/`)
   - Has `order` (number, >= 1)
   - If `loomId` present: must match `/^[a-f0-9]{32}$/`
   - If `loomEmbedUrl` present: must contain `loom.com/embed/`
   - Video orders within section are sequential starting from 1

5. **Completeness checks:**
   - Total video count matches `totalVideos` field
   - Number of sections is close to 13 (warn if < 10 or > 20, fail if < 5)
   - Total videos is close to 80 (warn if < 50 or > 120, fail if < 20)
   - Check for duplicate `mdId` values across all videos â€” warn if found

6. **Loom ID availability:**
   - Count videos with non-null `loomId` vs total
   - If 0 videos have `loomId`: log that Loom enrichment is needed (this is expected if iframes weren't on the root page)
   - If some have `loomId`: log the count and percentage

7. **Output a summary report** to console:
   ```
   === Manifest Validation Report ===
   File: output/manifest.json
   Extracted at: <timestamp>
   Course URL: <url>

   Sections: <N> (expected ~13)
   Total videos: <N> (expected ~80)
   Videos with Loom ID: <N>/<total> (<percent>%)
   Videos with title: <N>/<total>

   Section breakdown:
     1. <section name> â€” <N> videos
     2. <section name> â€” <N> videos
     ...

   Warnings: <list or "None">
   Errors: <list or "None">

   Result: PASS / WARN / FAIL
   ```

8. **Exit codes:** 0 = PASS, 1 = FAIL (structural errors), 2 = WARN (counts off but structure valid)
