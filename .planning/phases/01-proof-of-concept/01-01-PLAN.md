**Step 2: Browser console extraction script**

Create `scripts/extract-skool.js` as a self-contained IIFE designed to be pasted into the browser console on the authenticated Skool classroom ROOT page (`https://www.skool.com/bullrun-millions-crypto-course-9312/classroom` with NO `?md=` parameter).

The script MUST:

1. **Collect all lesson links via DOM anchor scraping** (primary approach per research):
   ```javascript
   document.querySelectorAll('a[href*="md="]')
   ```
   Extract the `md` ID from each href using regex: `/[?&]md=([a-f0-9]{32})/`
   Deduplicate by md ID.

2. **Extract video title** from each anchor's text content (`.textContent.trim()`).

3. **Extract section name and ordering** from DOM structure:
   - Look for section/module headings that are ancestors or siblings of the lesson links.
   - Try these selectors in order for finding the section heading element:
     a. Nearest ancestor or preceding sibling with a heading tag (`h2`, `h3`, `h4`)
     b. Nearest `[data-testid]` ancestor that looks like a module container
     c. Walk up the DOM tree to find the closest structural container, then look for its first heading child
   - If no section heading found, use `"Unknown Section"` (per research: section extraction is "best effort" in Phase 1).
   - Assign section order based on the order sections first appear in the DOM (top to bottom).
   - Assign video order within each section based on DOM order.

4. **Attempt to find Loom embed URLs** on the classroom root page:
   - Check for iframes: `document.querySelectorAll('iframe[src*="loom.com/embed"]')`
   - If found, extract Loom video ID with regex: `/\/embed\/([a-f0-9]{32})/`
   - If NO iframes on root page (likely — iframes are usually only on individual lesson pages), leave `loomId` and `loomEmbedUrl` as `null` and log a message: "Loom URLs not found on root page. Run enrichment on individual lesson pages or inspect __NEXT_DATA__ per lesson."

5. **Build the manifest JSON** in this exact structure:
   ```json
   {
     "extractedAt": "<ISO timestamp>",
     "courseUrl": "<current page URL>",
     "totalVideos": <count>,
     "sections": [
       {
         "name": "<section name>",
         "order": <1-based>,
         "videos": [
           {
             "title": "<video title>",
             "loomId": "<32-char hex or null>",
             "loomEmbedUrl": "<full embed URL or null>",
             "order": <1-based within section>,
             "mdId": "<32-char hex lesson ID>"
           }
         ]
       }
     ]
   }
   ```

6. **Output to clipboard** using `copy(JSON.stringify(manifest, null, 2))`.

7. **Log summary to console**: total videos found, total sections found, how many have Loom IDs vs null, and instruction: "Paste clipboard contents into output/manifest.json".

8. **Handle the lazy-loading pitfall** (from research Pitfall 4): Add a prominent comment at the top of the script warning the user to EXPAND ALL SIDEBAR SECTIONS before running the script. If the count of links found is suspiciously low (< 30), log a WARNING: "Only found N lessons. Make sure all sidebar sections are expanded before running."

**Important anti-patterns to avoid:**
- Do NOT rely on CSS class names like `[class*="module"]` — they are minified/generated (research Pitfall 5)
- Do NOT assume `__NEXT_DATA__` on root page has all videos (research Pitfall 6) — use DOM scraping as primary
- The script must work as a single paste into the browser console — no imports, no external dependencies

<task type="auto">
  <name>Task 2: Create Node.js Loom GraphQL transcript test script</name>
  <files>scripts/test-loom-api.mjs</files>
  <action>
Create `scripts/test-loom-api.mjs` — a standalone Node.js ESM script that validates the Loom GraphQL transcript API.

The script MUST:

1. **Accept a Loom video ID** as a command-line argument: `node scripts/test-loom-api.mjs <videoId>`
   Default to `13f9e28d4c434a878b8416bd8c364af3` (example from PROJECT.md) if no argument provided.

2. **POST to Loom GraphQL endpoint** exactly as documented in research:
   - URL: `https://www.loom.com/graphql`
   - Method: POST
   - Headers:
     - `Content-Type: application/json`
     - `Accept: application/json`
     - `User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36`
   - Body: Plain object (NOT array — research Pitfall 1):
     ```json
     {
       "operationName": "FetchVideoTranscript",
       "variables": { "videoId": "<id>", "password": null },
       "query": "query FetchVideoTranscript($videoId: ID!, $password: String) { fetchVideoTranscript(videoId: $videoId, password: $password) { ... on VideoTranscriptDetails { captions_source_url source_url transcription_status __typename } ... on GenericError { message __typename } __typename } }"
     }
     ```
   - Do NOT include `x-loom-request-source` or `apollographql-client-version` headers (research Pitfall 2)

3. **Handle the GraphQL response:**
   - Access `json.data.fetchVideoTranscript` (NOT `json.data[0].data` — research anti-pattern)
   - Check `__typename`: if `GenericError`, log the error message and exit with code 1
   - Check `transcription_status`: log it. If not `"completed"`, warn but continue
   - Extract `captions_source_url`: if null, log error with transcription_status and exit with code 1

4. **Fetch the VTT file** from `captions_source_url`:
   - Simple GET request
   - Parse VTT to plain text using the verified pattern from research:
     ```javascript
     function parseVttToText(vtt) {
       return vtt.split('\n')
         .filter(line =>
           !line.includes('-->') &&
           line.trim() !== '' &&
           !line.match(/^\d+$/) &&
           !line.startsWith('WEBVTT')
         )
         .map(l => l.trim())
         .join(' ')
         .trim();
     }
     ```

5. **Output results:**
   - Log: "Video ID: {id}"
   - Log: "Transcription status: {status}"
   - Log: "Captions URL: {url}"
   - Log: "Transcript length: {chars} characters, {words} words"
   - Log: "First 500 characters:" followed by the first 500 chars of plain text
   - Exit with code 0 on success

6. **Error handling:**
   - HTTP errors: Log status code and response body, exit 1
   - Network errors: Log error message, exit 1
   - Empty transcript (VTT parsed to empty string): Log warning, exit 1

Use `node-fetch` v3 (ESM import: `import fetch from 'node-fetch'`).

**Important anti-patterns to avoid:**
- Do NOT send the GraphQL body as an array `[{}]` (research Pitfall 1)
- Do NOT include stale version headers (research Pitfall 2)
- Do NOT access `response.data[0].data` — use `response.data.data` (research anti-pattern)
