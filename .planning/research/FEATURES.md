## Feature Landscape

### Table Stakes (Users Expect These)

Features that must work. Without any of these, the pipeline produces no usable output.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| Skool classroom URL/section scraping | Can't extract anything without finding the videos first | LOW | Browser console script running in authenticated session; DOM traversal to find Loom embed iframes; need to handle dynamically-rendered sections |
| Loom transcript fetch per video | Core of the pipeline — no transcript = no knowledge base | LOW | Loom auto-transcripts exist on all plans, 50+ languages; accessible via undocumented GraphQL endpoint (third-party scrapers use this); official API does NOT expose transcripts |
| Per-video markdown file creation | Deliverable format — markdown is the output | LOW | One file per video; filename derived from title + section; stored in folder hierarchy matching course structure |
| Folder hierarchy matching course structure | Without structure, 80 files are ungrouped noise | LOW | 13 sections → 13 folders; video files nested inside; naming must be sortable (01-, 02- prefixes) |
| Video title + section metadata in each file | Without context, a transcript is unreadable | LOW | Sourced from DOM scraping; included as YAML frontmatter or markdown headers |
| Raw transcript included in each file | The transcript is the source material | LOW | Full text, preserving paragraph/speaker breaks if available; timestamps stripped or retained as anchors |
| Per-section executive summary | 18 Mindset videos need a synthesized summary to be actionable | MEDIUM | One summary file per section; generated by AI API from all transcripts in section |
| Master executive summary | Single entry point to the entire course | MEDIUM | One master file summarizing all sections; generated from section summaries, not raw transcripts |
| End-to-end automation | Manual copy-paste defeats the purpose of a pipeline | MEDIUM | Single script invocation; handles all 80 videos sequentially without human intervention |

### Differentiators (Competitive Advantage)

Features that elevate output from "transcript dump" to "actionable playbook." These are the core value-add over manually reading transcripts.

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| Trading-specific summary format | Generic summaries miss what matters to a trader; domain-specific prompts extract entry signals, position sizing rules, risk management rules | MEDIUM | Prompt engineering: tell AI to extract concrete rules, not paraphrase; format as "Rule: X", "When: Y", "Do: Z" |
| Quick-reference cheat sheets per section | Cheat sheets are used during live trading; summaries are read once | MEDIUM | Separate from the full summary; ultra-condensed; bulleted; no prose; max 1 page equivalent |
| Transcript cleanup / normalization | Auto-generated Loom transcripts have filler words, run-on sentences, no punctuation; cleaned text summarizes better and reads better | MEDIUM | Strip filler words ("um", "uh", "you know"); add sentence boundaries; normalize capitalization; regex + simple NLP (no heavy model needed) |
| Progress tracking + resume on failure | 80 videos at ~5s/transcript = possible failures; without checkpointing, a failure at video 79 means restarting | LOW | JSON state file tracking which video IDs are done; skip completed on rerun; idempotent writes |
| Rate limit handling with backoff | Loom GraphQL endpoint will rate-limit bulk requests; without handling, extraction dies mid-run | LOW | Exponential backoff; configurable delay between requests (default: 1-2s) |
| Key takeaways section per video | Raw transcript alone requires re-reading to find the point; extracted takeaways = immediate value | MEDIUM | AI-generated per video; 3-7 bullet points; focused on actionable insight, not description |
| Section index file | Without an index, navigating 13 folders requires ls; index provides one-click access | LOW | One README.md per section listing all video files with titles; auto-generated from file list |
| Master course index | Top-level navigation file for the entire knowledge base | LOW | Lists all sections with link to section index; generated last |

### Anti-Features (Commonly Requested, Often Problematic)

Features that seem useful but would add complexity disproportionate to the value for a one-time extraction pipeline.

| Feature | Why Requested | Why Problematic | Alternative |
|---------|---------------|-----------------|-------------|
| Video file downloading | "I want offline access to videos" | Storage cost (80 videos × ~500MB = 40GB+); Loom ToS concerns; out of scope per PROJECT.md; transcripts capture the knowledge | Transcripts-only approach; knowledge is in the words, not the video |
| Real-time sync / incremental update | "What if Dan adds new videos?" | Skool auth sessions expire; course is effectively static during bull run; adds stateful sync complexity (change detection, diff management) | One-time extraction; re-run manually if course updates significantly |
| Semantic search index | "I want to search across all videos" | Requires vector DB, embeddings, query interface — an entire second system; not needed for 80-video personal knowledge base | Grep/ripgrep works perfectly on 80 markdown files; OS-level search (Spotlight, fzf) handles remainder |
| Web UI or viewer | "A nice UI would make this more accessible" | Scope explosion; markdown renders fine in Obsidian, VS Code, Typora, GitHub; building a viewer is a separate product | Use any markdown reader; knowledge base is portable as-is |
| Speaker diarization / labeling | "Who is speaking?" | Dan's course videos are single-speaker; diarization adds complexity with zero value | N/A — single speaker format means raw transcript is already attributable |
| Multi-language translation | "Translate summaries to Spanish" | Course is in English for English-speaking audience; adds LLM cost and complexity with no stated need | AI can translate on demand from the markdown files if needed |
| Confidence scoring per transcript | "How accurate is the transcript?" | Loom transcripts are pre-processed; WER scoring requires ground truth; no ROI for personal use | Flag low-confidence transcripts via length anomalies (very short transcripts for long videos) |
| Automated Skool login / session management | "Make it fully headless" | Skool uses anti-bot measures; session management is fragile; authenticated browser session approach is more reliable and simpler | Run browser script in authenticated session (user is already logged in); pipe output to Node script |

## Feature Dependencies

```
[Skool URL/section scraping]
    └──produces──> [Video URL list + metadata]
                       └──required by──> [Loom transcript fetch per video]
                                             └──produces──> [Raw transcript per video]
                                                                └──required by──> [Per-video markdown file]
                                                                └──required by──> [Transcript cleanup]
                                                                                      └──enhances──> [Per-video markdown file]
                                                                └──required by──> [Key takeaways per video]
                                                                                      └──required by──> [Per-video markdown file]

[Per-video markdown files] (all in section)
    └──required by──> [Per-section executive summary]
                           └──required by──> [Master executive summary]

[Per-video markdown files] (all sections)
    └──required by──> [Section index file]
    └──required by──> [Master course index]

[Progress tracking]
    └──enhances──> [Loom transcript fetch per video] (resume on failure)
    └──enhances──> [Per-video markdown file] (skip already done)

[Rate limit handling]
    └──required by──> [Loom transcript fetch per video] (reliability at scale)

[Trading-specific summary format]
    └──enhances──> [Per-section executive summary]
    └──enhances──> [Master executive summary]
    └──enhances──> [Key takeaways per video]
```

### Dependency Notes

- **Skool scraping must precede everything:** The video URL list is the seed for all downstream steps. Without it, nothing runs.
- **Transcript fetch precedes markdown creation:** No transcript = empty file = useless.
- **All per-video files in a section must exist before section summary:** Section summary is generated from the collection of transcripts/takeaways.
- **All section summaries must exist before master summary:** Master summary synthesizes sections, not raw transcripts (keeps context window manageable).
- **Progress tracking enhances transcript fetch:** Not a hard dependency — pipeline works without it — but without it a single failure in 80 videos requires full restart.
- **Transcript cleanup enhances summarization quality:** Dirty transcripts with filler words produce lower-quality AI summaries. Clean first, summarize second.

## MVP Definition

### Launch With (v1)

Minimum to produce a usable knowledge base from the 80-video course.

- [ ] Skool classroom scraper (browser console script) — extracts video titles, section names, Loom embed URLs in correct order
- [ ] Loom transcript fetch (Node.js script) — hits GraphQL endpoint per video ID, returns plain text transcript
- [ ] Per-video markdown file generation — title, section, raw transcript written to correct folder hierarchy
- [ ] Per-section executive summary — AI API generates actionable summary from all transcripts in section
- [ ] Master executive summary — AI API generates from all section summaries
- [ ] Progress state file — JSON tracking which video IDs are done; enables resume on failure

### Add After Validation (v1.x)

Add once v1 proves the pipeline works end-to-end.

- [ ] Transcript cleanup/normalization — improves summary quality; add after seeing how raw Loom transcripts look in practice
- [ ] Key takeaways per video — high value; but requires AI API call per video (80 calls); add after confirming cost is acceptable
- [ ] Quick-reference cheat sheets per section — format refinement; add once base summaries are validated as useful
- [ ] Section index files + master index — navigation improvement; low complexity, add as polish step
- [ ] Rate limit handling with backoff — add if rate limiting is observed during v1 run

### Future Consideration (v2+)

Defer until the playbook has been used and gaps are identified.

- [ ] Trading-specific prompt refinement — iterative improvement based on how useful v1 summaries are
- [ ] Re-run automation for course updates — only if Dan adds significant new content post-extraction

## Competitor Feature Analysis

This is a custom internal pipeline, not a product competing on the open market. Comparison is against existing tools that solve parts of this problem.

| Feature | Apify Loom Scraper | loom-transcript-mcp | skool-loom-dl (Go) | Our Approach |
|---------|-------------------|---------------------|-------------------|--------------|
| Skool URL extraction | No | No | Yes (downloads videos) | Browser console script — auth-aware, no downloads |
| Loom transcript fetch | Yes (paid Apify) | Yes (MCP, public only) | No | Direct GraphQL fetch — free, authenticated session |
| Folder hierarchy output | No | No | No | Yes — course structure preserved |
| AI summarization | No | No | No | Yes — per-section + master |
| Key takeaways extraction | No | No | No | Yes — per video |
| Trading playbook format | No | No | No | Yes — domain-specific prompts |
| Progress/resume | No | No | No | Yes — JSON state file |
| End-to-end automation | Partial | No | Partial | Yes — single script invocation |
| Cost | Apify credits | Free | Free | AI API cost only |

**Key insight:** No existing tool does the full pipeline. Available tools solve transcript extraction OR course scraping, never both, and none produce AI-synthesized knowledge bases. This pipeline bridges the gap with a custom glue layer.
